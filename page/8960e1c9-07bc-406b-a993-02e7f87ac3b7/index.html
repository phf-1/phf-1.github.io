The definition of an Actor is refined.

An objective describes necessary conditions that an object should satisfy for the objective to be reached.

The objective is to implement a function f~: \N \to \N such that f(i) ≡ 2 \times i in Python and in C.

An result is an object that has been built so far to reach the objective.

A Python implementation:

def f(x): return x * 2

A C implementation:

#include <limits.h> int a = <something>; int x = <something>; // There may be a need to check for -1 for two's complement machines. // If one number is -1 and another is INT_MIN, multiplying them we get abs(INT_MIN) which is 1 higher than INT_MAX if (a == -1 && x == INT_MIN) // `a * x` can overflow if (x == -1 && a == INT_MIN) // `a * x` (or `a / x`) can overflow // general case if (x != 0 && a > INT_MAX / x) // `a * x` would overflow if (x != 0 && a < INT_MIN / x) // `a * x` would underflow

Arbitrary computations encapsulated by the actor that are used to — among other things — produce the result.

The result may be a PDF file that needs a compilation step from encapsulated source files. So, when the result should be produced, the actor may perform an computation e.g. make pdf.

The context is a record of the steps taken by the actor in order to reach its objective. It is encapsulated. A step defines at least an objective and a result. A step may be in one of several states:

  1. Means « yet to be done ».
  2. Means « a result is being built ».
  3. Means « waiting for an other actor response ».
  4. Means « a result has been built and it satisfies the objective ». (Often omitted to prevent noise.)
  5. Means « a result — if any — has been built and it does not satisfy the objective ».
 A result.

A message is an arbitrary immutable piece of data that the actor may process whenever it can — like reading an email from a mailbox.
